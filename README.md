# Documentaciones

Compilación de documentaciones para mi mismo. El objetivo es redactar una documentación para cada herramienta o tecnología que voy aprendiendo de forma que me de la confianza de retomar dicha herramienta cuando ya haya olvidado como usarla. El enfoque de las documentaciones es generar explicaciones para mí mismo muy detalladas de cosas que me haya costado trabajo resolver, recordarme a mi mismo exactamente lo que entendí de las causas y soluciones.

En un principio comencé redactando desde cero, un poco desorganizado y sin secuencia, a veces incluso incompleto. Pero a partir de un momento comencé a hacer uso de agentes de IA para auxliarme en la redacción, pero sin perder el enfoque original que es explicarme a mí mismo las cosas. Para esto iré construyendo un catálogo de prompts (en la carpeta `prompts`) para agentes IA que me ayuden a acelerar el proceso de la documentación.

En un principio estoy usando el IDE `Antigravity` de **Google** que tiene incorporado un agente IA con diferentes modelos completamente gratis.

En la secciones siguientes desglosaré el flujo de trabajo para las distintas actividades de documentación, como iniciar una nueva documentación, mejorar una documentación, etc.

## Iniciar una nueva documentación

* Crear un archivo `.md` vació para la nueva documentación.
* Usar alguno de los prompts del catálogo para generar una documentación base y bien estructurada. Los disponibles para esta tarea son:
    * `prompts/nueva-documentacion.md`
* Revisar la documentación generada e ir realizando pruebas y aclarar dudas para complementar la documentación con mis propios comentarios y explicaciones e ideas, al mismo tiempo que aplico las modificaciones necesarias para que la documentación sea correcta y funcional.
* Almacenar el prompt usado para generar la documentación base en la misma carpeta del archivo `.md` correspondiente.
